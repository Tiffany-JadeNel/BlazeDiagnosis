# Student Starter Tasks by Group

These starter tasks are suitable for the first training phase. Tasks should be assigned as GitHub issues.

## Software Engineering 1

1. Map the main system entities and relationships.
2. Review the vehicle job status workflow.
3. Improve one user story with acceptance criteria.
4. Identify which status changes should create customer notifications.
5. Add or improve glossary entries.

## Software Engineering 2

1. Compare MVP user stories with the current backend module map.
2. Review database schema gaps.
3. Draft API contract improvements for job cards.
4. Draft API contract improvements for quotes and quote approvals.
5. Review role-based access requirements for workshop users and customers.

## Software Development 1

1. Read and explain one existing component.
2. Create a simple status badge component.
3. Improve spacing on a small page section using Tailwind CSS.
4. Add a placeholder empty state for a list page.
5. Create a short manual test note for a page.

## Software Development 2

1. Trace one feature from page to component to API wrapper.
2. Create or improve a customer list page.
3. Create or improve a vehicle details page.
4. Add a job card detail section.
5. Add form validation notes or basic validation to a starter form.

## Cloud Administration

1. Create a local environment setup checklist.
2. Document required environment variables using placeholder values only.
3. Review Docker Compose services and write usage notes.
4. Draft a deployment readiness checklist.
5. Draft a PostgreSQL backup and restore plan.
6. Review repository secret safety.

## Cyber Security

1. Create a security checklist for the app.
2. Review customer-facing access risks.
3. Review role-based access control needs.
4. Check repository security hygiene.
5. Draft a simple threat model for quote approvals and customer tracking.
