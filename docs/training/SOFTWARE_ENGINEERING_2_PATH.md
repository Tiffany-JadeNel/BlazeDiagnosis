# Software Engineering 2 Learning Path

## Primary Goal

Contribute to architecture, backend planning, database modelling, API contracts, and higher-complexity implementation tasks under review.

## Study Focus

- TypeScript architecture
- Backend module structure
- Database schema design
- Prisma currently used in this repository
- Drizzle ORM only if migration is approved
- API contracts
- Role-based access control
- Service/repository patterns
- Code review and implementation planning

## Start With

1. `docs/backend-module-map.md`
2. `docs/database-schema.md`
3. `docs/api-contracts.md`
4. `docs/access-control.md`
5. `docs/technical/ORM_DECISION_NOTE.md`

## Suitable Starter Tasks

- Review current schema against MVP user stories.
- Propose missing tables and relationships.
- Draft API contracts for job cards, quotes, parts, and invoices.
- Review role-based access requirements.
- Implement or plan a small backend service under mentor review.
- Review pull requests from Software Development students.
