# Software Engineering 1 Learning Path

## Primary Goal

Build a strong understanding of how the Blaze Diagnostics system is structured and how business requirements become user stories, entities, APIs, and tasks.

## Study Focus

- GitHub workflow
- TypeScript basics
- System entities and relationships
- User stories and acceptance criteria
- Basic API concepts
- Basic database concepts
- Job status workflow
- Documentation and clear communication

## Start With

1. `docs/product/DOMAIN_GLOSSARY.md`
2. `docs/backlog/MVP_USER_STORIES.md`
3. `docs/database-schema.md`
4. `docs/api-contracts.md`
5. `docs/backlog/STUDENT_STARTER_TASKS_BY_GROUP.md`

## Suitable Starter Tasks

- Document main system entities.
- Review a user story and improve acceptance criteria.
- Map the job card workflow.
- Identify which events should notify customers.
- Write glossary entries for workshop terms.
- Test and document an existing frontend screen.
